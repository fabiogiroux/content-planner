## O que fazer

Atualizar Dockerfile para incluir Chromium e pacotes de fontes no Alpine.

## Acceptance Criteria

- [ ] Chromium instalado via apk (não via npm)
- [ ] PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser
- [ ] PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
- [ ] Fontes instaladas: ttf-freefont, ttf-liberation, ttf-dejavu, font-noto, font-noto-emoji
- [ ] Imagem Docker buildável sem erros
- [ ] Puppeteer consegue lançar Chromium dentro do container
- [ ] HTMLs com Inter/Poppins/Roboto renderizam corretamente (via font-noto)

## Pacotes apk a adicionar

```dockerfile
RUN apk add --no-cache \
    chromium \
    nss \
    freetype \
    freetype-dev \
    harfbuzz \
    ca-certificates \
    ttf-freefont \
    ttf-liberation \
    ttf-dejavu \
    font-noto \
    font-noto-emoji

ENV PUPPETEER_EXECUTABLE_PATH=/usr/bin/chromium-browser
ENV PUPPETEER_SKIP_CHROMIUM_DOWNLOAD=true
```

## Risco

font-noto no Alpine cobre Google Fonts genéricas (Inter, Poppins, Roboto via webfont CDN).
Se Claude Designer usar fonte exótica via @import CDN, Puppeteer vai baixar a fonte durante
networkidle0 — deve funcionar. Se falhar: fallback para system-ui visível no PNG.
