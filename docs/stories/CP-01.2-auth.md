# CP-01.2 — Autenticação por senha simples

**Status:** Ready  
**Epic:** CP-01

## O que fazer

Middleware Express que protege todas as rotas com senha via cookie de sessão.

## Acceptance Criteria

- [ ] `GET /login` serve página de login (campo senha)
- [ ] `POST /login` valida senha contra `APP_PASSWORD`
- [ ] Cookie de sessão criado com expiração 7 dias
- [ ] Todas as rotas `/api/*` protegidas (retornam 401 sem sessão)
- [ ] Todas as páginas HTML protegidas (redirect para /login)
- [ ] Exceções: `/login`, `/favicon.ico`, `/api/health` (se existir)
- [ ] Se `APP_PASSWORD` não setada → app não sobe (erro no startup)
- [ ] Senha errada → mensagem de erro na página de login

## Implementação

```javascript
// Dependências a instalar:
// npm install express-session

// Middleware:
app.use(session({
  secret: process.env.SESSION_SECRET || APP_PASSWORD,
  resave: false,
  saveUninitialized: false,
  cookie: { maxAge: 7 * 24 * 60 * 60 * 1000 } // 7 dias
}))

// Guard:
function requireAuth(req, res, next) {
  if (req.session.authenticated) return next()
  if (req.path.startsWith('/api/')) return res.status(401).json({error: 'Unauthorized'})
  res.redirect('/login')
}
```

## Design da página /login

- Padrão visual Warm Indigo (igual ao app)
- Campo único: senha
- Botão "Entrar"
- Mensagem de erro inline se senha errada
- Sem username — single-user por instância
